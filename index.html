QQ 群聊小助手

丰富的群聊功能，让你的群更加活跃有趣

安装指南

快速部署到你的服务器或本地环境

1

克隆项目

从 GitHub 克隆最新代码到本地

git clone https://github.com/codehyq/qq-bot.git
cd qq-bot
2

配置环境

配置网易云 Cookie（可选，但推荐配置以提升搜索准确度）

cd bot
echo "NETEASE_COOKIE=你的cookie" >> .env

从浏览器登录 music.163.com 提取 Cookie

3

启动服务

使用 Docker Compose 启动所有服务

docker compose up -d
4

配置 LLBot Desktop

下载并配置 QQ 机器人客户端

  1. 下载 LLBot Desktop，解压到固定目录
  2. 运行 llbot.exe，点击"启动"后用 QQ 小号扫码登录
  3. 打开 http://localhost:3080，配置反向 WebSocket

WebSocket 地址: ws://localhost:8080/onebot/v11/ws

AccessToken: (留空)

5

测试

在群里发送 @机器人 帮助 查看所有功能

架构设计 

QQ群消息 → LLBot Desktop (QQ NT 宿主)
           │ 反向 WebSocket (OneBot v11)
           ▼
      NoneBot2 (Python, :8080)
      ├── at_bot.py       # @机器人拦截器
      ├── music.py        # 点歌
      ├── music_extra.py  # 歌词/每日一歌/猜歌
      ├── idiom_chain.py  # 成语接龙
      ├── weather.py      # 天气
      ├── quote.py       # 一言
      ├── fortune.py      # 抽签
      ├── choose.py       # 选择困难症
      ├── daily_lang.py   # 每日英语/日语
      ├── image.py        # 美图
      ├── help.py         # 帮助菜单
      └── db.py          # SQLite 数据库
           │
      ┌────┴────┐
      ▼         ▼
 musicapi   qqmusicapi
(Netease)  (QQ Music)
 :13000     :13300
  • LLBot Desktop 运行在 Windows，整合 QQ NT 和 OneBot 协议
  • NoneBot 和 MusicAPI 在 Docker 中运行
  • 反向 WebSocket 连接，无需暴露公网端口

插件开发

添加新功能只需简单几步

1 在 bot/plugins/ 下新建 .py 文件
2 导入 help_registry 并调用 register()
3 使用 on_command() 或 on_message() 注册处理器
4 NoneBot 自动加载，无需修改 bot.py

注意: 修改代码后执行 docker compose restart nonebot 生效

常见问题

解决你可能遇到的常见问题

确认 Docker 后端已启动，运行 docker ps 查看 qq-bot-nonebot-1 是否为 Up 状态。检查 LLBot 的反向 WebSocket 配置是否正确，地址应为 ws://localhost:8080/onebot/v11/ws。
这是 QQ NT 桌面端的限制，网易云音乐卡片在桌面端显示"暂不支持的消息类型"。建议使用手机 QQ 尝试，音乐卡片在手机端可以正常点击播放。
网易云音乐搜索依赖 Cookie，请从浏览器登录 music.163.com 后提取 Cookie，写入 bot/.env 文件的 NETEASE_COOKIE 配置项。Cookie 过期后需要重新提取。
这是正常现象。LLBot 接管了 QQ 连接后，官方 QQ 客户端会被挤下线。使用小号进行测试可以避免影响主要账号。
在 bot/plugins/ 目录下创建新的 .py 文件，导入 help_registry 并调用 register() 注册帮助信息，使用 on_command() 或 on_message() 注册命令处理器。NoneBot 会自动加载，无需修改 bot.py。改代码后执行 docker compose restart nonebot 即可生效。
3000、8081 等低位端口可能被系统占用。修改 docker-compose.yml 中的端口映射，使用高位端口（如 13000:3000）来解决此问题。
